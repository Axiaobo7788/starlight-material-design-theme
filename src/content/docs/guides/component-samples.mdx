The examples keep Starlight's original component model in place while the theme
adds token-driven color, shape, state, and spacing.

## Asides

<Aside type="note" title="Surface note">
	Notes use a neutral elevated container so they read as supporting context instead of a loud alert.
</Aside>

<Aside type="tip" title="Primary tip">
	Tips use the primary container role and should feel helpful, not promotional.
</Aside>

<Aside type="caution" title="Secondary caution">
	Caution blocks use warm tonal roles, with a restrained outline instead of a heavy vertical stripe.
</Aside>

<Aside type="danger" title="Error container">
	Danger blocks use error container roles and should remain readable in both light and dark modes.
</Aside>

## Steps

<Steps>
1. Define Material-style `--md-sys-*` tokens.
2. Bridge those tokens into the current Starlight `--sl-*` variables.
3. Style high-impact documentation surfaces first: sidebar, header, prose, code, asides, cards.
4. Add component overrides only when a consuming site needs a markup-level change.
</Steps>

## Code

Inline `code` should be legible without becoming a bright pill in the paragraph.

```ts
import starlight from '@astrojs/starlight';
import md3Theme from 'starlight-theme-md3';

export default {
	integrations: [
		starlight({
			plugins: [md3Theme()],
		}),
	],
};
```

## Table

| Surface | MD3 role | Starlight target |
| --- | --- | --- |
| Page shell | `surface-container-lowest` | `--sl-color-bg` |
| Main content | `surface` | `main > .content-panel` |
| Header | `surface-container` | `--sl-color-bg-nav` |
| Sidebar active item | `secondary-container` | `[aria-current='page']` |
| Code block | `surface-container-low` | Markdown `pre` |
| Aside tip | `primary-container` | `.starlight-aside--tip` |
